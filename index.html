Project

Introduction

One day a manager had the idea to create a website to help people in IT technologies. The site allows users to ask questions through threads. Others can respond to the questions and vote for the best answers.

Because information technologies are very diverse, the manager wants to assign tags to each thread to classify and easily find them.

Pages

Once the site is completed, it will consist of the following pages:

  • Home
    This page contains the welcome message from the manager for visitors and highlights the “hottest” thread of the website.
  • Thread list
    This page provides a list containing threads created by users. The search by name may also be possible. Each item in the list consists of the thread name as well as the first characters of the question.
  • Thread details
    This page is used to display the thread selected, including the name, the tags, comments left by users and a system rating for each one. The page also contains a form to leave a comment.

Tasks

Marking Schema

For each achieved you get one point

Task 1

Design two tables: the comment table and the associative table between thread and tag.

  • A thread can have multiple tags
  • A tag can have multiple threads
  • A comment is associated with a thread
  • The first comment is the question itself
  • Comment fields: name and email of the author, a rating (up and down) represented by a number, and the content of the comment

Task 2

Read the “hottest” thread from the database and display it on the home page.

  • The thread with the highest rating total is displayed (up minus down)
  • Use the template included in the home page

Task 3

Implement the comment system on a thread according to the description in Task 1.

  • The list is shown on the thread list page
    • Each thread comment is cropped to a maximum of 200 chars
  • The detail is shown on the thread detail page
    • The first comment is the thread
    • All comments assigned to a thread are shown
  • A form o add a thread is shown on the thread list page
    • Add the form with the fields name, email, thread name, comment
    • Check the format of the email address.
    • Handle errors that may occur (blank, invalid email, etc.).
  • A form o add a comment to a thread is shown on the thread detail page
    • Add the form with the fields name, email and comment
    • Check the format of the email address.
    • Handle errors that may occur (blank, invalid email, etc.).

Implement the rating system. Each comment can be rated up or down by the user. The rating sums (plus and minus) are shown below the comment with rating links.

Task 5

Create a form which allows to assign tags to a thread on the tread detail page. Show the tag list below the thread name.

  • Check if the tag already exists in the database when adding a new one:
    • if it exists, link the tag to the thread.
    • if not, create the new tag and link it to the thread
  • Make the tag submission case insensitive (convert to lowercase when you check for tags)
  • Use the bootstrap “label” CSS class (search in documentation) for the tags displayed
  • Alert a success message - examples you can find in the design page
